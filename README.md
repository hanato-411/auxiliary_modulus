This repository contains the training and dataset-generation workflow for modular-addition experiments.

- `train/train.py`

- `train/hf_transformer.py`

  - `experiment_entrypoints.md`
  - `sweep_parameter_guide.md`

## Docker Setup

```bash
make build
make run
```

## Experiment Workflow (Docker)

### 1. Generate Datasets

Use these generation configs:

- `config/generate/group_a_small.yaml`
- `config/generate/group_b_mid.yaml`
- `config/generate/group_c_large.yaml`

```bash
python3 -m scripts.generate_dataset --config config/generate/group_a_small.yaml
python3 -m scripts.generate_dataset --config config/generate/group_b_mid.yaml
python3 -m scripts.generate_dataset --config config/generate/group_c_large.yaml
```

See `docs/dataset_generation_settings.md` for parameter details.

### 2. Single-Config Training

All single-run training settings are managed in `config/config_train.yaml`.
`train/train.py` loads this file at runtime.

```bash
docker exec -it auxiliary_modular_addition-container bash -lc \
  'cd /app && sage train/train.py'
```

See `docs/train_mod_settings.md` for training configuration details.

### 3. Sweep Experiments

Create experiment-specific sweep files from `config/sweep/templates/template_sweep.yaml`,
then run:

```bash
wandb sweep <your-sweep-config.yaml>
wandb agent <entity/project/sweep_id>
```

Reproducibility-focused sweep configs are planned under `sweep/experiments`.
At the moment, reproduction is already possible by editing `config/sweep/templates/template_sweep.yaml` and running it as a custom sweep config.

See `docs/experiment_entrypoints.md` and `docs/sweep_parameter_guide.md` for usage guidance.

## Notes

- Dataset files are generated by `scripts/generate_dataset.py` using `calt`'s `DatasetGenerator`.
- Stop and remove the container with `make stop` when finished.
